# DeskVane 平台功能矩阵

本文档对应 [跨平台重构计划](./cross-platform-refactor-plan.md) 中的 Phase 0 产出物，用来定义三平台的功能边界、MVP 范围和发布形态。

## 平台分级

| 平台 | 当前支持级别 | 发布形态 | 说明 |
| --- | --- | --- | --- |
| Linux（Debian / Ubuntu） | 正式支持 | `.deb`、`pipx` | 现阶段主平台，功能最完整。 |
| Windows | 开发中 | 计划提供安装器 / `.exe` | 当前以源码兼容和 CI 验证为主，未发布正式安装包。 |
| macOS | 开发中 | 计划提供 `.app` / `.dmg` | 当前以源码兼容和 CI 验证为主，未发布正式安装包。 |

## 功能矩阵

说明：
- `正式支持`：已经纳入当前平台范围，目标是可稳定使用。
- `MVP`：属于该平台首批需要完成的核心能力。
- `规划中`：已明确方向，但尚未交付。
- `Linux-only`：当前明确只在 Linux 提供。

| 功能 | Linux | Windows | macOS | 备注 |
| --- | --- | --- | --- | --- |
| 托盘 | 正式支持 | MVP / 开发中 | MVP / 开发中 | Phase 1-3 已完成共享菜单模型拆分。 |
| 设置面板 | 正式支持 | MVP / 开发中 | MVP / 开发中 | Tk UI 共享，平台差异主要在系统集成。 |
| 截图 | 正式支持 | MVP / 开发中 | MVP / 开发中 | 已抽成 controller/service，等待平台实现补齐。 |
| 钉图 | 正式支持 | MVP / 开发中 | MVP / 开发中 | 依赖截图和 Tk 叠窗行为验证。 |
| 剪贴板历史 | 正式支持 | MVP / 开发中 | MVP / 开发中 | 文本剪贴板已纳入平台服务层。 |
| 翻译模块 | 正式支持 | MVP / 开发中 | MVP / 开发中 | 默认关闭，按需启用。 |

## 平台 MVP

### Linux

- 保持现有完整能力。
- 继续作为主发布平台。
- 现有 `.deb` 打包链路保持可用。

### Windows

- 托盘
- 设置
- 截图
- 钉图
- 剪贴板历史
- 翻译模块
- Git 代理

### macOS

- 托盘
- 设置
- 截图
- 钉图
- 剪贴板历史
- 翻译模块

## 发布形态

| 平台 | 目标发布形态 |
| --- | --- |
| Linux（Debian / Ubuntu） | `.deb` |
| Linux（开发者） | `pipx install .` |
| Windows | 安装器 / `.exe` |
| macOS | `.app` / `.dmg` |

## 当前阶段结论

- Linux 已具备正式发布路径。
- Windows 和 macOS 当前的重点不是“直接可发包”，而是先完成平台服务层、共享逻辑拆分、功能 gating 和 CI 基础验证。
- 在发布 Windows/macOS 安装包之前，README 和支持声明必须继续明确：这两个平台目前仍处于开发中状态。
