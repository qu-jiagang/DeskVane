# DeskVane 平台支持声明

本文档说明 DeskVane 当前对各平台的正式支持范围，以及尚未承诺的内容。

## 当前正式支持

当前正式支持的平台是：

- Linux（Debian / Ubuntu）

对该平台的承诺包括：

- 提供正式安装路径：`.deb`
- 提供开发者安装路径：`pipx install .`
- 保持托盘、截图、剪贴板、翻译、代理和设置面板的完整体验
- 修复正式支持范围内的兼容性问题

## 开发中平台

以下平台已纳入跨平台重构目标，但当前仍处于开发中状态：

- Windows
- macOS

当前阶段的承诺仅包括：

- 平台服务层持续抽象
- Linux-only 能力显式隔离
- 在 GitHub Actions 中进行基础导入和核心单元测试验证
- Windows / macOS 保持基础平台实现持续可回归

当前阶段不承诺：

- 提供正式安装包
- 完整功能对齐
- 与 Linux 相同的系统集成能力

## Linux-only 功能声明

以下功能当前明确仅在 Linux 提供：

- 终端代理环境继承

这些功能在非 Linux 平台上应满足两条要求：

- UI 不暴露误导性的入口
- 配置会自动归一到受支持的实现

## 支持等级说明

| 等级 | 含义 |
| --- | --- |
| 正式支持 | 有安装说明、发布路径和问题修复承诺。 |
| 开发中 | 进入重构和 CI 覆盖范围，但尚未承诺可供终端用户直接使用。 |
| 不支持 | 当前不在目标范围内，或明确不计划在近期实现。 |

## 后续升级条件

Windows 或 macOS 要升级为“正式支持”，至少需要满足：

1. 具备稳定的平台注册实现，而不是仅靠 Null fallback。
2. 核心 MVP 功能可用：托盘、截图、钉图、剪贴板历史、翻译。
3. 有对应平台的打包和安装说明。
4. 三平台 CI 覆盖核心导入、核心单测和最小 smoke test。

在这些条件满足之前，Windows 和 macOS 应继续以“开发中”对外说明。

当前进度补充：

- Windows 已具备独立的 tray / hotkey / autostart 平台实现，以及构建入口
- macOS 已具备独立的 tray / hotkey / autostart 平台实现，以及 `.app` / `.dmg` 构建入口
- 两个平台仍未完成正式发布前要求的实机验收、签名和分发流程
